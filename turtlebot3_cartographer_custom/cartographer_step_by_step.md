# 🗺️ การเริ่มทำ Cartographer SLAM แบบ Step-by-Step

การใช้หุ่นยนต์สำรวจห้องเพื่อสร้างแผนที่แบบ 2 มิติด้วยระบบ **Cartographer** ในโปรเจกต์นี้ จะแบ่งเป็นหลักการ "เปิดซิม -> บังคับสำรวจ -> เซฟแผนที่" โดยสามารถทำตามลำดับได้ดังนี้:

## Step 1: เปิดพื้นที่จำลอง (Simulator) ในโหมด SLAM
เปิด Terminal แล้วรัน Launch ของหุ่นยนต์โดยเพิ่ม Option `slam:=true` ติดเข้าไปด้วย (ห้ามรันคู่กับ `nav:=true`)

**ตัวอย่างรัน CustomBot (1 คัน):**
```bash
ros2 launch webots_diffdrive custom_bot_launch.launch.py slam:=true
```
*(ระบบจะเปิดโปรแกรม Webots และจอ RViz2 ขึ้นมา ซึ่งคุณจะเห็นรอยตีกรอบรอบๆ หุ่นยนต์จาก LiDAR สีฟ้าๆ เริ่มปรากฏขึ้น)*

---

## Step 2: บังคับหุ่นกวาดแผนที่ (Teleop)
เพื่อให้แผนที่สมบูรณ์ หุ่นต้องวิ่งไปสแกนด้วย LiDAR ให้ครบทุกซอกมุม 
ให้เปิด Terminal **หน้าต่างใหม่** แล้วใช้คำสั่งบังคับเดินมาตรฐานของ ROS 2:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/robot1/cmd_vel
```
- **ควบคุมด้วยคีย์บอร์ด:** `U, I, O / J, K, L / M, ,, .` ในการกดเดิน
- **Tips:** ค่อยๆ พาหุ่นวิ่งลัดเลาะขอบกำแพงช้าๆ ไม่ต้องรีบหักเลี้ยวรุนแรง หากมีมุมอับให้ค่อยๆ หมุนตัวกล้องกวาด เมื่อเรามองในหน้าต่าง RViz จะเห็นหมอกมืดค่อยๆ ทยอยเปิดเห็นแพทเทิร์นกำแพงห้อง (สีดำ) อย่างชัดเจน

---

## Step 3: บันทึกแผนที่เก็บไว้ (Save Map)
เมื่อตรวจสอบผ่านโปรแกรม RViz แล้วว่าแผนที่ทั้งหมดต่อกันสวยงามและเดินครบห้องแล้ว เราจำเป็นต้องดึงแผนที่นั้นออกมาจากวงจร (Dump) ให้เปิด Terminal **หน้าต่างใหม่**:

```bash
ros2 run nav2_map_server map_saver_cli -f my_new_map --ros-args -r map:=/robot1/map
```
*(หมายเหตุ: ต้องแก้ตรง `/robot1/map` ให้สัมพันธ์กับชื่อ Namespace หุ่นยนต์ที่คุณรัน ถ้ามีหุ่นหลายตัวรันพร้อมกัน)*

**ไฟล์ที่ได้รับหลังเซฟสำเร็จ:**
คำสั่งนี้จะทำการเซฟไฟล์ 2 นามสกุลในตำแหน่งที่คุณกำลังพิมพ์คำสั่ง (ปัจจุบันคือ Workspace):
1. `my_new_map.pgm` (รุปภาพแปลนอาคาร ขาว-ดำ-เทา)
2. `my_new_map.yaml` (ไฟล์โค้ดระบุการตั้งค่าสเกลต่างๆ)

---

## Step 4: จบงานและแพ็กไปใช้กับ Navigation
ที่ Terminal ทุกๆ ไซต์ ให้กดคีย์ชอร์ตคัต `Ctrl + C` ปิดระบบ Webots และโปรแกรมให้หมด!
หลังจากนี้ หากคุณต้องการให้หุ่นเดินอัตโนมัติ คุณก็นำพิกัดไฟล์ `my_new_map.yaml` ที่ได้ ไปตั้งค่าแทนที่อันเก่าในโฟลเดอร์ `maps/` แล้วสลับไปรันแบบโหมด `nav:=true` เพื่อทดสอบให้หุ่นวิ่งหาแผนที่นี้แบบ Auto ได้เลยครับ
